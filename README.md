QR-Hotel — Цифрова система для гостьових номерів

Репозиторій містить Специфікацію вимог до програмного забезпечення (Software Requirements Specification — SRS) для інформаційного сервісу гостьового самообслуговування через QR-коди.

Крок 3. Специфікація вимог (SRS)

Вступ
1.1 Мета документа: Цей документ визначає функціональні та нефункціональні вимоги до системи QR-Hotel, що призначена для заміни застарілих паперових інформаційних буклетів у готельних номерах на інтерактивний цифровий довідник.
1.2 Сфера застосування: Система охоплює веб-интерфейс для гостей готелю (доступний без реєстрації через сканування QR-коду в номері) та панель адміністратора для керування контентом (інформація про готель, замовлення послуг, меню рум-сервісу, контакти).

Загальний опис системи
Перспектива продукту: Автономна веб-система, що працює на будь-яких мобільних пристроях через браузер.
Актори системи (Користувачі):

Гість (Незареєстрований користувач) — сканує QR-код, переглядає довідник кімнати, меню ресторану, замовляє послуги (прибирання, таксі, рум-сервіс).

Адміністратор готелю — авторизований працівник, який редагує інформацію, оновлює ціни, змінює пароль від Wi-Fi, керує активними замовленнями в реальному часі.

Функціональні вимоги (Functional Requirements)
FR-01: Система повинна генерувати унікальний QR-код для кожної кімнати готелю.
FR-02: Система повинна відображати цифровий гід готелю (правила проживання, внутрішні номери, інструкції).
FR-03: Система повинна містити актуальний пароль від Wi-Fi на головному екрані гостя.
FR-04: Система повинна відображати інтерактивне меню рум-сервісу (доставка їжі в номер) з цінами та фото.
FR-05: Система повинна дозволяти гостю викликати персонал (кнопка Прибирання номера або Потрібен рушник).
FR-06: Система повинна надавати форму для замовлення додаткових послуг (бронювання сауни, трансфер, виклик таксі).
FR-07: Система повинна підтримувати мультимовність (мінімум українська та англійська мови інтерфейсу).
FR-08: Панель адміністратора повинна вимагати авторизацію (логін та пароль) для доступу.
FR-09: Адміністратор повинен мати можливість миттєво змінювати текстові дані (наприклад, новий пароль Wi-Fi) без перезапуску системи.
FR-10: Адміністратор повинен бачити список нових запитів від кімнат у вигляді звукових або візуальних сповіщень.
FR-11: Система повинна автоматично надсилати підтвердження замовлення їжі на екран гостя після прийняття його адміністратором.

Нефункціональні вимоги (Non-Functional Requirements)
NFR-01 (Швидкість): Час завантаження будь-якої сторінки гостя на мобільному пристрої при 4G-зв'язку не повинен перевищувати 2 секунд.
NFR-02 (Адаптивність): Веб-інтерфейс повинен бути на 100% адаптованим під мобільні екрани будь-якого розширення (iOS, Android).
NFR-03 (Доступність): Система повинна працювати стабільно в режимі 24/7 з показником доступності не менше 99.5%.
NFR-04 (Безпека даних): Усі з'єднання з панеллю адміністратора та передача замовлень повинні бути захищені за протоколом HTTPS.

Обмеження (Constraints)
Система працює виключно за наявності активного інтернет-з'єднання.
Проєкт реалізується як веб-додаток (PWA), тому встановлення окремого мобільного додатка з App Store чи Google Play не передбачене.

Глосарій
QR-код (Quick Response Code) — матричний двовимірний штрих-код, який містить пряме посилання на сторінку конкретного номера готелю.
Рум-сервіс (Room Service) — послуга доставки їжі, напоїв або речей безпосередньо в готельну кімнату користувача.
Специфікація вимог (SRS) — документ, який детально описує всі можливості та обмеження програмного продукту
